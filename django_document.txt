
WEEK 5 — FORMS & USER INPUT (SUPER SIMPLE VERSION)
✅ 1. What is a Form?
Simple Definition:
A form is a box where users type information and send it to your website.

Example:
Login form

Registration form

Contact form

✅ 2. Types of Forms in Django
(A) Form
You create the input fields yourself.

(B) ModelForm
Django creates the input fields automatically from a model.

✅ 3. Simple Django Form Example
forms.py
from django import forms

class ContactForm(forms.Form):
    name = forms.CharField()
    email = forms.EmailField()
    message = forms.CharField()
Meaning:
CharField() → normal text

EmailField() → must be a valid email

✅ 4. Showing the Form in a View
views.py
from django.shortcuts import render
from .forms import ContactForm

def contact_view(request):
    form = ContactForm()  # empty form

    if request.method == "POST":
        form = ContactForm(request.POST)  # form with user input
        if form.is_valid():  # if input is correct
            print(form.cleaned_data)
            return render(request, "done.html")

    return render(request, "contact.html", {"form": form})
✅ 5. HTML Form (Very Simple)
contact.html
<form method="POST">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">Send</button>
</form>

✅What is CSRF Token?
Simple Definition:
A special security code that protects your form from attacks.

Example:
{% csrf_token %}
Put it inside every <form>.

✅What is Validation?
Validation means “checking if the user entered correct 
information.”

✅ Example:
1. Email must look like: example@gmail.com
2. Age must be a number
3. Name cannot be empty

Django checks this when you use:
form.is_valid()

✅ What is cleaned_data?
cleaned_data is the safe and correct data after validation.

✅ Example:
name = form.cleaned_data["name"]

✅ ModelForm (Easier Form)
ModelForm automatically creates a form from a database model.

✅ Example
✅model.py
class Student(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()

✅ModelForm
from django import forms
from .models import Student

class StudentForm(forms.ModelForm):
    class Meta:
        model = Student
        fields = ["name", "age"]

✅view.py
def register(request):
    form = StudentForm()

    if request.method == "POST":
        form = StudentForm(request.POST)
        if form.is_valid():
            form.save()   # saves to database
            return render(request, "done.html")

    return render(request, "register.html", {"form": form})

✅ SUMMARY
1. Form: A box where users type information.
2. ModelForm: Form created automatically from a model.
3. Validation: Django checks if data is correct.
4. is_valid(): True when all inputs follow rules.
5. cleaned_data: The correct data you can use.
6. CSRF token: Protects your form from hackers.
7. form.save(): Saves ModelForm data into the database.

AUTHENTICATION & MESSAGES (VERY SIMPLE VERSION)
✅ What is Authentication?
it means checking if a user is who they say they are

✅Example
1. Login
2. Logout
3. Register (create account)

Note: Django already has these features built-in

PART ONE - USER REGISTRATION

✅Registering a New User - Registration means creating a
new account 

✅Example:
 forms.py

from django import forms
from django.contrib.auth.models import User

class RegisterForm(forms.ModelForm):
    password = forms.CharField(widget=forms.PasswordInput)

    class Meta:
        model = User
        fields = ["username", "email", "password"]


views.py
from django.shortcuts import render, redirect
from .forms import RegisterForm
from django.contrib.auth.models import User

def register_view(request):
    form = RegisterForm()

    if request.method == "POST":
        form = RegisterForm(request.POST)
        if form.is_valid():
            user = form.save(commit=False)
            user.set_password(form.cleaned_data["password"])
            user.save()
            return redirect("login")

    return render(request, "register.html", {"form": form})


PART 2 - USER LOGIN

✅Login means checking username  + password and letting the 
user into the website

✅views.py
  from django.contrib.auth import authenticate, login

def login_view(request):
    if request.method == "POST":
        username = request.POST["username"]
        password = request.POST["password"]

        user = authenticate(username=username, password=password)

        if user is not None:
            login(request, user)
            return redirect("dashboard")  # after login
        else:
            return render(request, "login.html", {"error": "Invalid login"})

    return render(request, "login.html")


PART 3 - USER LOGOUT
✅ Logging out - means ending the user's session

✅Example
from django.contrib.auth import logout

def logout_view(request):
    logout(request)
    return redirect("login")

PART 4 - RESTRICTING PAGES
✅ Restricting Pages - means that some pages should only open 
if the user is logged in, using "@login_required"

✅Example
from django.contrib.auth.decorators import login_required

@login_required
def dashboard(request):
    return render(request, "dashboard.html")

if the user is NOT logged in: They are redirected to the 
login page


PART 5 - DJANGO MESSAGES
Django messages allow you to show short notifications to the 
user

✅Example
1. "Registration successful"
2. "Login failed"
3. "You have logged out."

✅ How to use Messages (adding it to the view)
if user is not None: 	   
    login(request, user)
    messages.success(request, "Login successful!")
    return redirect("dashboard")
else:
    messages.error(request, "Invalid username or password")


✅ Showing Messages in HTML (Adding it to the base template):
{% if messages %}
    {% for msg in messages %}
        <p>{{ msg }}</p>
    {% endfor %}
{% endif %}
